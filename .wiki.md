# Project Summary
This project is a digital health application designed to assist patients in managing back pain through personalized exercise and nutrition plans. It integrates front-end and back-end services to deliver a seamless user experience, including patient authentication, progress tracking, and access to tailored health content. The application empowers users to take charge of their health, improving their quality of life, especially in managing back pain. Recent updates have introduced a strict weekly progress requirement for users to maintain access to their programs.

# Project Module Description
The project consists of two primary modules:
- **Backend**: Manages patient authentication, data management, and API services.
- **Frontend**: Provides a user-friendly interface for patients to interact with health programs, including logging in, tracking exercises, and accessing resources.

# File Description Inventory
- **Backend**
  - `main.py`: Entry point for the backend application.
  - `routers/`: API route definitions for patient authentication and program management.
  - `services/`: Business logic for managing patient data, exercises, and weekly progress.
  - `data_models/`: JSON files defining patient and program data structures.
  - `mock_data/`: Mock data files for testing purposes.
  - `models/patients.py`: Defines the patient data model and related functionalities.
  - `models/weekly_progress.py`: Defines the weekly progress data model.
  - `routers/patient_auth.py`: Handles patient authentication routes.
  - `routers/weekly_progress.py`: Manages weekly progress entries.
  - `services/patient_sync.py`: Manages synchronization of patient data.
  - `services/weekly_progress.py`: Contains business logic for weekly progress entries.

- **Frontend**
  - `src/`: Contains React components, pages, and contexts for managing application state.
    - `pages/`: Specific pages of the application including `Index.tsx`, `SavaitesProgresas.tsx`, and `Pratimai.tsx`.
    - `contexts/`: Manages patient context for authentication state.
  - `public/`: Static assets like images and favicon.
  - `package.json`: Lists dependencies and scripts for running the frontend application.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy, PostgreSQL
- **Frontend**: React, TypeScript, Vite, Tailwind CSS

# Usage
To set up and run the project:
1. **Install Dependencies**:
   - For Backend: Navigate to the backend directory and run `pip install -r requirements.txt`.
   - For Frontend: Navigate to the frontend directory and run `pnpm install`.

2. **Build the Project**:
   - For Frontend: Run `pnpm run build`.

3. **Run the Application**:
   - Start the backend server and frontend development server as per your setup instructions.
